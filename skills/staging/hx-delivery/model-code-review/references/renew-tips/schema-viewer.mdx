## Introduction

The **Schema Viewer** is a Custom UI Component (CUIC) designed to help developers inspect and interact with the values of a data schema **after they have been calculated from the rating algorithm**. It provides an expandable tree-view of all schema nodes—**inputs, outputs, and overrides**—allowing you to easily drill into nested structures and monitor value changes in real time.

This tool is especially useful for **debugging** complex models, as it helps visualize the final state of the schema in a clear and interactive format.

Use the Schema Viewer to:

- **Visualize schema structure**: See field names and current values at a glance.  
- **Live-update monitoring**: Automatically refresh node values as your rating algorithm calculates them.  
- **Export schema as JSON**: Download the full schema in JSON format, which can be plugged back into **hx Renew** for further investigation or reproducibility.

## Limitations

The Schema Viewer currently does not support two specific node types: `hx.File` and `hx.Triangle`. Any nodes of these types will be omitted from the rendered schema tree and will not appear in the viewer.

## Setup

To try out the Schema Viewer, you can download the files [here](../assets/schema_viewer_model.zip) and plug into your model for debugging.  

### Algorithm

To enable the Schema Viewer, you’ll need to plug a utility function into your rating algorithm. This function collects the structured schema data and prepares it for display.

Here’s the required setup:

In `rating.py`:

```python
from algorithms.schema_view.rating import schema_view_rating

@hx.rating
def rating_algorithm(hxd):
    # Main rating algorithm here...
    schema_view_rating(hxd=hxd)
```

The function `schema_view_rating` will **recursively transform the data schema into JSON format**, making it suitable for inspection and export.

> **Important:** The `hxd_to_json` function must be called at the **end** of the rating algorithm. This ensures the schema is fully populated and includes all calculated values.

### Data Schema

To support the Schema Viewer, your data schema must include a node named `stringified_json`. This node is used to store the **JSON-encoded version of the entire data schema**, as generated by the `schema_view_rating` function.

Here’s an example of how to include it in `data_schema.py`:

```python
import hx_data_schema as hx
from data_schema.schema_view import schema_view

@hx.data_schema
def data_schema():
    return hx.Structure(
        children={
            # Other Structures...
            **schema_view()
        }
    )
```

The `schema_view()` function adds both the `stringified_json` node and a `show_view` flag, which controls whether the viewer is shown in the UI.

### View

To display the Schema Viewer in the frontend UI, you need to include the component `{SchemaViewPage()}` in your view structure. This component renders a collapsible panel that reads from the `schema_view/stringified_json` node and is conditionally shown based on the `schema_view/show_view` flag.

Here’s an example of how to include it in `view.js`:

```javascript
import * as HX from "hx-model-components";
import SchemaViewPage from "view/schema_view";

function HXModel() {
  return (
    <HX.Root>
      // Other Pages...
      {SchemaViewPage()}
    </HX.Root>
  );
}
```

This ensures the viewer is conditionally rendered when the component is enabled for inspection.
