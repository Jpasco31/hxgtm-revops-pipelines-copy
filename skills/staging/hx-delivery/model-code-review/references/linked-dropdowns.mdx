## Updating the parameter table

Currently, the model contains a parameter table called **'industry'** which contains a list of industries and their corresponding base rates. This needs to be replaced with a new table called `industry_trade`, which contains the fields required for the linked dropdown.

**Download the industry_trade.csv [here](../assets/industry_trade.csv)**

> **Note:** Remember to update the `parameter_tables_schema.json` file to reflect the changes above.

For more information about parameter tables, please see the [Developer Documentation](/platform/developer/building-your-model/algorithms/parameter-tables#defining-parameter-tablesx).

## Updating the data schema - flat dropdown

The next step is to update the data schema by creating a structure to store the contents of the linked dropdown. Initially this will be a flat dropdown, later this will be modified to be a hierarchical dropdown.

1. In the data schema, delete the `industry` node and in its place create a new **input list** called `industries`.
2. Inside the industries list create a new **structure** called `industry_trade` with the following props `linked_options_table = "industry_trade`, `linked_options_columns =["industry", "trade"]`, `linked_default_index = 0`.

   This will allow the new parameter table to be read into the model.

3. Add two **input string** nodes within `industry_trade`, these should be called `industry` and `trade` and should have appropriate labels.
4. Next, create four additional nodes within `industries`, but outside of the `industry_trade` structure. These should be: `allocation`, `base_rate`, `exposure` and `premium`. Format the nodes appropriately based on the spreadsheet model:

<img src={require("../assets/linked_dropdown_nodes.png").default} width="800" />

5. Finally, outside of `industries` list create another new structure called `industry_totals`, containing a node called `allocation` which will be used to **output** the total allocation.

To view the code for the updates to the data schema, expand the pane below.

<details>
<summary>Data schema solution - flat linked dropdown</summary>

```python
  "industries": hx.List(mode="input", children={
            "industry_trade" : hx.Structure(linked_options_table="industry_trade", linked_options_columns=["industry", "trade"], linked_default_index=0, children={
                "industry": hx.Str(mode="input", view={"label":"Industry"}),
                "trade": hx.Str(mode="input", view={"label": "Trade"}),
            }),
            "allocation": hx.Float(mode="input", default=0, view={"label": "Allocation", "format": {"output" : "percent", "mantissa":1}}),
            "base_rate": hx.Float(mode="override", view={"label": "Base Rate", "format": {"output" : "percent", "mantissa":2}}),
            "exposure": hx.Float(mode="output", view={"label": "Exposure", "format": {"thousandSeparated":True, "mantissa":0}}),
            "premium": hx.Float(mode="output", view={"label": "Premium", "format": {"thousandSeparated":True, "mantissa":0}}),
        }),
  "industry_totals": hx.Structure(children={
            "allocation": hx.Float(mode="output", view={"label": "Allocation", "format":{"output":"percent", "mantissa":2}}),
    }),

```

</details>

## Updating the rating algorithm

It is now time to update `rating.py`.

1. The first change to be made is to ensure that the algorithm is reading `industry_trade` rather than `industry`.

2. Next, the algorithm will needs to loop through the industries list and lookup the model base rate using the both the industry and trade fields.

The general syntax for this is below:

`param_table[(param_table["col_1"] == val_1) & (param_table["col_2"] == val_2)]["col_3"].iloc[0]`

> **Hints:**
>
> - The bounds for the loop should be **`for industry in hxd.industries:`**
> - Inside the loop three things should happen:
>   - `industry.base_rate.calculated` should set to the base rate of the industry that has been selected.
>   - `industry.exposure` needs to be calculated (exposure \* allocation)
>   - `industry.premium` should also be calculated (exposure \* base rate)

3. Outside the loop, the allocation total needs to be calculated, this is done using the following code:

```python
 hxd.industry_totals.allocation = sum([industry.allocation for industry in hxd.industries if industry.allocation is not None])
```

4. Add a validation error `(hx.errors.validation)` to ensure the allocations sum to 1. Numpy will need to be imported at the top of the code.

> **Note:** for further information on error handling, please see the [Developer Documentation](/platform/developer/building-your-model/algorithms/error-handling#validation-errors)

5. Update the base premium calculation to be:

```python
 hxd.base_premium = sum([industry.premium for industry in hxd.industries if industry.premium is not None])
```

6. Finally, update any other references to the old model base rate. There are references in the rating algorithm and the example data.

> **Hint:** The code in step 3 will help when calculating the base premium.

Check your code against the solution below.

<details>

<summary>Rating algorithm solution</summary>

```python
    import math

    # 1 Read industry_trade
    industry_trade_table = hx.params.industry_trade

    # 2 Loop to find base rate by industry
    for industry in hxd.industries:
        industry.base_rate.calculated = industry_trade_table[(industry_trade_table["industry"] == industry.industry_trade.industry) & (industry_trade_table["trade"] == industry.industry_trade.trade)]["rate"].iloc[0]
        industry.exposure = hxd.exposure.value * industry.allocation
        industry.premium = industry.exposure * industry.base_rate.selected

    # 3 Setting allocation total
    hxd.industry_totals.allocation = sum([industry.allocation for industry in hxd.industries if industry.allocation is not None])

    # 4 Add a validation error to ensure the allocations sum to 1
    if not math.isclose(hxd.industry_totals.allocation, 1, rel_tol=0.001):
            hx.errors.validation("Industry allocations must sum to 1")

    # 5 Base premium calculations
    hxd.base_premium = sum([industry.premium for industry in hxd.industries if industry.premium is not None])
    #hxd.base_premium = hxd.exposure.value * hxd.model_base_rate.selected
```

</details>

## Hierarchical linked dropdown

In order to convert the dropdown to be hierarchical opposed to flat a change needs to be made to the data schema. A **view** argument can be added to the `industry_trade` structure with the `linked_options_selector` property set to `hierarchical`.

```python
 "industries": hx.List(mode="input", children={
            "industry_trade": hx.Structure(view={"linked_options_selector": "hierarchical"}, linked_options_table="industry_trade", linked_options_columns=["industry", "trade"], linked_default_index=0, children={
                "industry_name": hx.Str(mode="input", view={"label": "Industry"}),
                "trade": hx.Str(mode="input", view={"label": "Trade"}),
            }),
             "allocation": hx.Float(mode="input", default=0, view={"label": "Allocation", "format": {"output": "percent", "mantissa": 1}}),
    })
```
