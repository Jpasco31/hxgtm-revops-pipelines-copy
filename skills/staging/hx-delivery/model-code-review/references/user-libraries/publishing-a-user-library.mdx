## Labelling User Library Versions

In order to make a version of a user library live and available for importing to other models and user libraries, you must first identify the user library version by assigning it a label.

1. Open the library that you wish to publish

<img
  src={require("../../assets/ModelDevelopmentProcess/Models/libraries_add_label_1.png").default}
  width="1000"
/>

2. Click on the branch that you wish to publish

<img
  src={require("../../assets/ModelDevelopmentProcess/Models/libraries_add_label_2.png").default}
  width="1000"
/>

3. Click on the 'more actions' button alongside the version that you want to add a label to, then select 'Set Label'

<img
  src={require("../../assets/ModelDevelopmentProcess/Models/libraries_set_label.png").default}
  width="1000"
/>

3. Give your version a label which will be used to identify the version; for example, a version number. This label will be visible to users, so ensure you choose something which will be meaningful to them

<img
  src={require("../../assets/ModelDevelopmentProcess/Models/libraries_set_label_2.png").default}
  width="600"
/>

## Making a Version of a User Library Live

To make a user library available for import, you need to make a labelled version of the user library live:

1. Click on any labelled version of a user library, in this example we will use the 'Initial version'.

<img
  src={require("../../assets/ModelDevelopmentProcess/Models/libraries_set_live_1.png").default}
/>

2. Click on toggle 'Live' to make the version live

<img
  src={require("../../assets/ModelDevelopmentProcess/Models/libraries_set_live_2.png").default}
  width="1000"
/>

## Disabling a Version of a User Library for Import

To make a user library not available for import, click on toggle 'Live' again. When the user library is disabled, it is not automatically removed from other models and/or user libraries which have already imported it; it is only made unavailable for new imports.
